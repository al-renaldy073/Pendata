Data Understanding#

Pengertian Data Understanding#

Data Understanding adalah tahap dalam proses Data Mining yang bertujuan untuk memahami karakteristik, struktur, dan kualitas data yang akan digunakan dalam analisis.

Setelah pada tahap sebelumnya kita memahami masalah bisnis, maka pada tahap ini kita mulai memahami:

Data apa yang tersedia dan apakah data tersebut mampu menjawab masalah bisnis?

Tahap ini menjadi jembatan antara kebutuhan bisnis dan proses teknis analisis data.

Tujuan Data Understanding#

Tujuan utama tahap ini adalah:

  1. Mengumpulkan data yang relevan

  2. Mengenali struktur dan tipe data

  3. Menilai kualitas data

  4. Mengidentifikasi pola awal

  5. Menemukan potensi masalah dalam data

Tahap ini membantu memastikan bahwa data benar-benar sesuai dengan tujuan proyek.

Mengapa Data Understanding Penting?#

Tanpa memahami data dengan baik:

  • Model bisa dibangun dari data yang salah

  • Analisis menjadi tidak akurat

  • Kesimpulan bisa menyesatkan

Contoh: Jika ingin memprediksi churn pelanggan tetapi data tidak memiliki informasi riwayat transaksi, maka prediksi menjadi kurang akurat.

Tahapan dalam Data Understanding#

  1. Pengumpulan Data (Data Collection) Data dapat berasal dari berbagai sumber, seperti:

    • Database internal perusahaan

    • File CSV atau Excel

    • API

    • Data publik

    • Sistem transaksi

    Pada tahap ini, penting untuk memastikan bahwa data yang dikumpulkan relevan dengan tujuan bisnis.

  2. Deskripsi Data (Describing Data) Langkah berikutnya adalah memahami struktur data.

    Hal-hal yang diperiksa:

    • Jumlah baris dan kolom

    • Nama variabel

    • Tipe data (numerik, kategorikal, teks, tanggal)

    • Rentang nilai setiap variabel

    Contoh:

    • Kolom “Umur” bertipe numerik

    • Kolom “Jenis Kelamin” bertipe kategorikal

    • Kolom “Tanggal Transaksi” bertipe datetime

  3. Exploratory Data Analysis (EDA) Exploratory Data Analysis (EDA) bertujuan untuk menemukan pola awal dalam data.

    Kegiatan yang dilakukan:

    • Menghitung statistik deskriptif (mean, median, standar deviasi)

    • Melihat distribusi data (histogram)

    • Menganalisis hubungan antar variabel (korelasi)

    • Mengidentifikasi outlier

    EDA membantu memahami karakteristik data sebelum masuk ke tahap pembersihan dan pemodelan.

  4. Evaluasi Kualitas Data Data dunia nyata sering memiliki masalah, seperti:

    • Missing values (nilai kosong)

    • Duplikasi data

    • Kesalahan input

    • Outlier ekstrem

    • Inkonsistensi format

    Pada tahap ini, semua potensi masalah tersebut diidentifikasi untuk diperbaiki pada tahap Data Preparation.

Contoh Sederhana#

Misalkan kita memiliki data pelanggan dengan kolom:

  • ID Pelanggan

  • Umur

  • Pendapatan

  • Jumlah Transaksi

  • Status Churn

Pada tahap Data Understanding, kita:

  1. Menghitung rata-rata umur

  2. Melihat distribusi pendapatan

  3. Mengecek apakah ada nilai kosong

  4. Melihat hubungan antara jumlah transaksi dan status churn

Dari sini kita mulai mendapatkan insight awal.

Hubungan dengan Tahap Lain#

Data Understanding berada di antara:

Business Understanding \(\rightarrow\) Data Understanding \(\rightarrow\) Data Preparation

Jika pada tahap ini ditemukan bahwa data tidak lengkap atau tidak relevan, maka kita mungkin perlu:

  • Mengumpulkan data tambahan

  • Mengubah tujuan analisis

  • Kembali ke tahap Business Understanding

Proses ini bersifat iteratif.

Kesalahan Umum dalam Data Understanding#

  1. Tidak melakukan eksplorasi data secara menyeluruh

  2. Langsung masuk ke modelling tanpa EDA

  3. Mengabaikan missing values

  4. Tidak memeriksa distribusi data

Kesimpulan#

Data Understanding adalah tahap penting dalam proses Data Mining yang bertujuan untuk memahami struktur, kualitas, dan pola dalam data. Tahap ini memastikan bahwa data yang digunakan benar-benar sesuai untuk menjawab masalah bisnis dan siap untuk diproses lebih lanjut pada tahap Data Preparation.

Eksplorasi data iris#

Eksplorasi data merupakan tahap awal dalam analisis data yang bertujuan untuk memahami karakteristik dataset sebelum dilakukan pemodelan. Pada dataset Iris, proses eksplorasi dilakukan untuk mengetahui pola, distribusi data, hubungan antar variabel, serta ringkasan statistik dari data bunga iris.

Dataset Iris terdiri dari 150 data bunga dengan 4 atribut numerik:

  • Sepal Length (panjang kelopak)

  • Sepal Width (lebar kelopak)

  • Petal Length (panjang mahkota)

  • Petal Width (lebar mahkota)

Serta 1 atribut kategorikal:

  • Species (jenis bunga: Setosa, Versicolor, Virginica)

Penjelasan Fitur Secara Mendalam#

Semua fitur bertipe:

Numerical (Continuous Float)
Skala pengukuran: Rasio (Ratio scale)
Satuan: Centimeter (cm)

  1. Sepal Length (cm)

    • Panjang kelopak bunga bagian luar

    • Range: ± 4.3 – 7.9 cm

    • Makna biologis: menunjukkan ukuran bunga secara umum

    Rata-rata keseluruhan: ≈ 5.84 cm

  2. Sepal Width (cm)

    • Lebar kelopak luar

    • Range: ± 2.0 – 4.4 cm

    • Fitur ini sering overlap antar class

    Rata-rata: ≈ 3.05 cm

  3. Petal Length (cm)

    • Panjang mahkota bunga

    • Range: ± 1.0 – 6.9 cm

    • Fitur PALING membedakan class

    Rata-rata: ≈ 3.76 cm

  4. Petal Width (cm)

    • Lebar mahkota bunga

    • Range: ± 0.1 – 2.5 cm

    • Sangat kuat untuk klasifikasi

    Rata-rata: ≈ 1.20 cm

Visualisasi Data#

A. Tujuan Visualisasi#

Visualisasi digunakan untuk:

  • Melihat distribusi data

  • Mengidentifikasi pola

  • Mengetahui pemisahan antar kelas

  • Mendeteksi outlier

  • Mempermudah interpretasi data

B. Visualisasi menggunakan Orange#

Pada Orange, visualisasi dilakukan dengan workflow berikut:

SQL Table \(\rightarrow\) Data Table \(\rightarrow\) Scatter Plot/Select Columns\(\rightarrow\)Rank/Pivot Table/Correlations/Box Plot/Column Statistics/Distributions

Beberapa visualisasi penting yang di coba:

  1. Sql Table

    Widget Sql Table merupakan pengaturan koneksi database pada widget SQL Table di aplikasi Orange untuk mengambil data dari PostgreSQL..

    Bagian Server

    • PostgreSQL : Jenis database yang digunakan.

    • localhost : Server database berada di komputer sendiri (lokal).

    • Pendata : Nama database yang akan diakses.

    • postgres : Username untuk login ke database.

    • Password (••••••) : Kata sandi akun database.

    • Connect : Tombol untuk menghubungkan Orange ke database PostgreSQL.

    Bagian Data Selection

    • Table: (iris) Memilih tabel bernama iris dari database untuk digunakan sebagai dataset.

    • Custom SQL : Opsi untuk menulis query SQL sendiri jika ingin mengambil data dengan perintah tertentu.

    • Auto-discover categorical variables : Jika dicentang, Orange otomatis mendeteksi kolom kategorikal (misalnya kolom species).

  2. Data Table

    Widget Data Table digunakan untuk menampilkan dataset dalam bentuk tabel, sehingga pengguna dapat melihat isi data secara lengkap.

  3. Scatter Plot

    Widget Scatter Plot digunakan untuk menampilkan hubungan antara dua atribut dalam bentuk grafik titik, sehingga dapat melihat pola dan penyebaran data.

    Contoh: hubungan antara sepal length dan sepal width, sehingga kita dapat melihat pola penyebaran data dan perbedaan antar kelas bunga Iris.

  4. Select Columns \(\rightarrow\) Rank

    Widget Select Columns digunakan untuk memilih atribut yang akan digunakan dan menentukan variabel target. Widget Rank digunakan untuk menilai dan memberi peringkat atribut berdasarkan tingkat kepentingannya terhadap variabel target.

    • Select Columns

    • Rank

    Widget Select Columns digunakan untuk memilih atribut yang akan digunakan dan menentukan variabel target, sedangkan widget Rank digunakan untuk menilai dan memberi peringkat atribut berdasarkan tingkat kepentingannya terhadap variabel target.

  5. Pivot Table

    Widget Pivot Table digunakan untuk merangkum dan menghitung data berdasarkan kategori tertentu dalam bentuk tabel ringkasan. Pada tampilan ini, Pivot Table digunakan untuk menghitung jumlah data pada setiap kelas species. Hasilnya menunjukkan bahwa setiap kelas Iris (setosa, versicolor, virginica) memiliki masing-masing 50 data, dengan total 150 data.

  6. Correlations

    Widget Correlations digunakan untuk menghitung dan menampilkan nilai korelasi antar atribut menggunakan metode Pearson correlation.

    Dari hasil terlihat bahwa petal length dan petal width memiliki korelasi sangat kuat sebesar +0.963, artinya kedua atribut memiliki hubungan yang sangat erat. Sedangkan beberapa atribut lain memiliki korelasi sedang hingga lemah, bahkan negatif.

  7. Box Plot Widget Box Plot digunakan untuk menampilkan distribusi data, nilai median, rentang, dan outlier dari suatu atribut dalam bentuk box plot.

    Pada tampilan ini, Box Plot digunakan untuk membandingkan distribusi sepal width pada setiap kelas species. Terlihat bahwa setiap kelas memiliki median dan rentang nilai yang berbeda, sehingga membantu memahami perbedaan karakteristik antar kelas Iris.

  8. Column Statistics

    Widget Column Statistics digunakan untuk menampilkan statistik deskriptif dari setiap atribut, seperti mean, median, minimum, maksimum, dan distribusi data.

    Widget ini membantu untuk memahami karakteristik data secara statistik, misalnya rata-rata sepal length adalah 5.843, serta melihat penyebaran dan rentang nilai setiap atribut pada dataset Iris.

  9. Distributions

    Widget Distributions digunakan untuk menampilkan distribusi atau penyebaran nilai suatu atribut dalam bentuk histogram.

    Pada tampilan ini, histogram menunjukkan distribusi nilai petal width, sehingga kita dapat melihat frekuensi kemunculan nilai tertentu dan memahami pola penyebaran data.

Analisa korelasi#

A. Tujuan Analisa Korelasi#

Analisis korelasi digunakan untuk mengetahui:

  • Seberapa kuat hubungan antar variabel

  • Atribut mana yang paling berpengaruh

  • Redundansi fitur

  • Dasar feature selection

Nilai korelasi berada pada rentang -1 sampai 1:

  • Mendekati 1 \(\rightarrow\) hubungan positif kuat

  • Mendekati -1 \(\rightarrow\) hubungan negatif kuat

  • Mendekati 0 \(\rightarrow\) tidak ada hubungan

B. Korelasi menggunakan Orange#

Workflow: File \(\rightarrow\) Correlations

Hasil dataset Iris Untuk Analisa Korelasi:

  • Petal Length \(\leftrightarrow\) Petal Width \(\rightarrow\) korelasi sangat tinggi

  • Sepal Length \(\leftrightarrow\) Petal Length \(\rightarrow\) korelasi tinggi

  • Sepal Width \(\rightarrow\) korelasi relatif rendah dengan atribut lain

Interpretasi: Petal attributes merupakan fitur paling penting dalam membedakan spesies.

Penjelasan hal tersebut:

  1. Tabel matriks korelasi Menampilkan semua pasangan variabel dalam bentuk matriks. Contoh:

    • petal_length vs petal_width = 0.962757

    • petal_length vs sepal_length = 0.871754

    • Nilai tampil sampai 6 desimal.

  2. Widget Correlations di Orange Menampilkan pasangan variabel saja (pairwise) dalam bentuk daftar. Biasanya diurutkan dari korelasi terbesar \(\rightarrow\) terkecil. Nilai ditampilkan dalam bentuk pembulatan (3 desimal):

    • 0.962757 \(\rightarrow\) 0.963

    • 0.871754 \(\rightarrow\) 0.872

Jadi bukan beda hasil, tetapi: Metode sama \(\rightarrow\) Pearson correlation

  • Data sama

  • Nilai sama

  • Hanya beda format tampilan & pembulatan

  • Standar deviasi

  • Dll.

C. Statistik Deskriptif menggunakan Python#

Kode berikut digunakan untuk menghitung statistik deskriptif pada dataset IRIS, yang bertujuan untuk memberikan gambaran umum mengenai karakteristik data, seperti jumlah data, rata-rata, nilai minimum, maksimum, kuartil, standar deviasi, variansi, skewness, dan modus.

df_numeric = df.select_dtypes(include=['int64','float64'])

for col in df_numeric.columns:
    print("Statistik untuk", col,)
    print("Jumlah Data      :", df[col].count())
    print("Rata-rata        :", df[col].mean())
    print("Nilai Minimum    :", df[col].min())
    print("Q1               :", df[col].quantile(0.25))
    print("Median (Q2)      :", df[col].quantile(0.5))
    print("Q3               :", df[col].quantile(0.75))
    print("Nilai Maximum    :", df[col].max())
    print("Kemencengan      :", round(df[col].skew(), 6))
    mode = df[col].mode()
    print("Modus            :", mode.iloc[0] if not mode.empty else "Tidak ada modus")
    print("Standar Deviasi  :", round(df[col].std(), 6))
    print("Variansi         :", round(df[col].var(), 6))
    print("\n")

Cek Outlier

Outlier adalah data yang memiliki nilai ekstrem dan berbeda jauh dari mayoritas data lainnya.

Pada dataset Iris, outlier bisa muncul jika ada nilai ukuran bunga (misalnya panjang sepal atau petal) yang terlalu besar atau terlalu kecil dibandingkan data lainnya.

Cara Menentukan Outlier pada Iris

Biasanya digunakan metode IQR (Interquartile Range): $\(IQR = Q_3 - Q_1\)$

Batas bawah: $\(\text{Batas Bawah} = Q_1 - 1.5 \times IQR\)$

Batas atas: $\(\text{Batas Atas} = Q_3 + 1.5 \times IQR\)$

Apakah Dataset Iris Memiliki Outlier? Secara umum:

  • Dataset Iris relatif bersih

  • Tidak banyak outlier ekstrem

  • Namun pada beberapa fitur seperti sepal width, kadang muncul beberapa nilai yang terdeteksi sebagai outlier berdasarkan metode IQR

Tetapi outlier tersebut masih termasuk variasi alami ukuran bunga, bukan kesalahan data.

Kesimpulan Outlier:

Outlier pada dataset Iris adalah nilai ukuran bunga yang berada jauh di luar rentang mayoritas data. Berdasarkan metode IQR, dataset Iris umumnya tidak memiliki outlier ekstrem yang signifikan, sehingga dataset ini tergolong bersih dan cocok digunakan untuk analisis atau machine learning.

outlier_summary = []
for col in numeric_cols:
    Q1 = df[col].quantile(0.25)
    Q3 = df[col].quantile(0.75)
    IQR = Q3 - Q1
    lower = Q1 - 1.5 * IQR
    upper = Q3 + 1.5 * IQR
    jumlah_outlier = df[(df[col] < lower) | (df[col] > upper)].shape[0]
    outlier_summary.append([col, jumlah_outlier])
outlier_table = pd.DataFrame(outlier_summary, columns=["Kolom", "Jumlah Outlier"])

Penjelasan Kode:

  • outlier_summary = [] digunakan untuk membuat list kosong yang akan menyimpan ringkasan jumlah outlier setiap kolom numerik.

  • for col in numeric_cols: digunakan untuk melakukan perulangan pada setiap kolom numerik yang terdapat dalam variabel numeric_cols.

  • Q1 = df[col].quantile(0.25) digunakan untuk menghitung kuartil pertama (25%) dari kolom tersebut.

  • Q3 = df[col].quantile(0.75) digunakan untuk menghitung kuartil ketiga (75%) dari kolom tersebut.

  • IQR = Q3 - Q1 digunakan untuk menghitung Interquartile Range (IQR), yaitu selisih antara Q3 dan Q1.

  • lower = Q1 - 1.5 * IQR digunakan untuk menentukan batas bawah outlier berdasarkan metode IQR.

  • upper = Q3 + 1.5 * IQR digunakan untuk menentukan batas atas outlier berdasarkan metode IQR.

  • jumlah_outlier = df[(df[col] < lower) | (df[col] > upper)].shape[0] digunakan untuk menghitung jumlah data yang berada di luar batas bawah atau batas atas (outlier).

    • (df[col] < lower) | (df[col] > upper) adalah kondisi untuk memilih data yang termasuk outlier.

    • .shape[0] digunakan untuk mengambil jumlah baris (banyaknya outlier).

  • outlier_summary.append([col, jumlah_outlier]) digunakan untuk menyimpan nama kolom dan jumlah outlier ke dalam list outlier_summary.

  • outlier_table = pd.DataFrame(outlier_summary, columns=[“Kolom”, “Jumlah Outlier”]) digunakan untuk mengubah list ringkasan menjadi DataFrame dengan dua kolom, yaitu nama kolom dan jumlah outlier.

  • outlier_table dipanggil untuk menampilkan tabel jumlah outlier pada setiap kolom numerik.

Berdasarkan analisis outlier menggunakan metode IQR pada dataset Iris, mayoritas data berada dalam rentang normal untuk setiap kolom numerik. Dari empat kolom numerik (sepal_length, sepal_width, petal_length, petal_width), hanya kolom sepal_width yang memiliki sejumlah kecil nilai outlier, sementara kolom lainnya relatif bersih tanpa outlier. Outlier pada sepal_width muncul di beberapa baris dan ditandai dalam tabel, namun jumlahnya sangat kecil dibandingkan total 150 baris, sehingga tidak signifikan untuk memengaruhi analisis keseluruhan. Secara keseluruhan, dataset Iris tergolong bersih dan stabil, sehingga sebagian besar model statistik atau machine learning dapat diterapkan tanpa perlu penanganan outlier yang besar.

outlier_rows = pd.DataFrame()
for col in numeric_cols:
    Q1 = df[col].quantile(0.25)
    Q3 = df[col].quantile(0.75)
    IQR = Q3 - Q1
    lower = Q1 - 1.5 * IQR
    upper = Q3 + 1.5 * IQR
    temp = df[(df[col] < lower) | (df[col] > upper)]
    outlier_rows = pd.concat([outlier_rows, temp])
outlier_rows = outlier_rows.drop_duplicates()

Penjelasan Kode:

  • outlier_rows = pd.DataFrame() digunakan untuk membuat DataFrame kosong yang akan menyimpan seluruh baris data yang terdeteksi sebagai outlier.

  • for col in numeric_cols: digunakan untuk melakukan perulangan pada setiap kolom numerik dalam dataset.

  • Q1 = df[col].quantile(0.25) digunakan untuk menghitung kuartil pertama (25%) dari kolom tersebut.

  • Q3 = df[col].quantile(0.75) digunakan untuk menghitung kuartil ketiga (75%) dari kolom tersebut.

  • IQR = Q3 - Q1 digunakan untuk menghitung Interquartile Range (IQR), yaitu selisih antara Q3 dan Q1.

  • lower = Q1 - 1.5 * IQR digunakan untuk menentukan batas bawah outlier berdasarkan metode IQR.

  • upper = Q3 + 1.5 * IQR digunakan untuk menentukan batas atas outlier berdasarkan metode IQR.

  • temp = df[(df[col] < lower) | (df[col] > upper)] digunakan untuk mengambil baris data yang memiliki nilai lebih kecil dari batas bawah atau lebih besar dari batas atas (termasuk outlier pada kolom tersebut).

  • outlier_rows = pd.concat([outlier_rows, temp]) digunakan untuk menggabungkan (menambahkan) baris outlier yang ditemukan ke dalam DataFrame outlier_rows.

  • outlier_rows = outlier_rows.drop_duplicates() digunakan untuk menghapus baris yang duplikat, karena satu baris bisa terdeteksi sebagai outlier di lebih dari satu kolom.

  • outlier_rows dipanggil untuk menampilkan seluruh baris data yang termasuk outlier pada minimal satu kolom numerik.

Dari dataset Iris yang berisi 150 baris, analisis outlier menunjukkan bahwa mayoritas data berada dalam rentang normal. Hanya beberapa baris yang memiliki nilai outlier, dan semuanya muncul di kolom sepal_width, sedangkan kolom lainnya (sepal_length, petal_length, petal_width) relatif bersih tanpa outlier.

Jumlah baris yang terdeteksi sebagai outlier sangat kecil dibandingkan total dataset, sehingga pengaruhnya terhadap analisis keseluruhan sangat minimal. Secara keseluruhan, dataset ini tergolong bersih dan stabil, sehingga sebagian besar analisis statistik atau model machine learning dapat diterapkan tanpa penanganan outlier yang signifikan.

Menghitung Jarak Dataset Iris#

Pada tahap awal analisis, dataset yang digunakan adalah Iris Dataset yang diakses melalui database PostgreSQL. Dataset ini berisi 150 data bunga iris dengan empat atribut numerik, yaitu sepal length, sepal width, petal length, dan petal width, serta satu atribut kelas yang menunjukkan jenis bunga iris.

Proses analisis dilakukan menggunakan Orange Data Mining dengan memanfaatkan beberapa widget utama yaitu Dataset, Distance, dan Distance Matrix.

A. Tujuan Menghitung Jarak Dataset Iris#

Visualisasi digunakan untuk:

  • Melihat tingkat kemiripan atau perbedaan antar data pada Iris Dataset

  • Mengidentifikasi pola hubungan antar data berdasarkan nilai jarak yang dihasilkan dari Euclidean Distance

  • Mengetahui kelompok data yang memiliki karakteristik yang mirip

  • Membantu mendeteksi data yang memiliki jarak sangat jauh dari data lainnya

  • Mempermudah interpretasi hasil perhitungan jarak dalam bentuk matriks

B. Visualisasi Menghitung Jarak menggunakan Orange#

Pada Orange, visualisasi dilakukan dengan workflow berikut:

SQL Table \(\rightarrow\) Distances \(\rightarrow\) Distance Matrix

Beberapa visualisasi penting yang di coba:

  1. Sql Table

    Widget SQL Table merupakan pengaturan koneksi database pada widget SQL Table di aplikasi Orange Data Mining untuk mengambil data dari database PostgreSQL. Melalui widget ini, pengguna dapat menghubungkan Orange dengan database sehingga dataset dapat diakses secara langsung tanpa perlu mengimpor file secara manual. Pada penelitian ini, widget SQL Table digunakan untuk mengambil data dari Iris Dataset yang tersimpan dalam database PostgreSQL.

  2. Distances Widget Distances digunakan untuk menghitung jarak atau tingkat kemiripan antar data dalam dataset. Pada analisis ini digunakan metode Euclidean Distance untuk mengukur jarak antar objek data.

    Pada widget Distances terdapat beberapa pengaturan penting, yaitu:

    1. Distance Metric Pada analisis ini dipilih metode Euclidean Distance karena atribut pada dataset Iris berupa data numerik sehingga metode ini cocok digunakan untuk menghitung jarak antar objek data.

    2. Compare Pengaturan Compare digunakan untuk menentukan jenis objek yang akan dibandingkan. Pada analisis ini dipilih Rows, yang berarti perhitungan jarak dilakukan antar baris data. Dengan kata lain, setiap baris data bunga iris akan dibandingkan dengan baris data lainnya untuk mengetahui tingkat kemiripannya.

    Euclidean Distance menghitung jarak antara dua titik data berdasarkan perbedaan nilai atribut numerik yang dimiliki oleh setiap data. Pada dataset Iris, atribut yang digunakan dalam perhitungan jarak meliputi sepal length, sepal width, petal length, dan petal width.

    Dengan metode ini, setiap data akan dibandingkan dengan data lainnya sehingga menghasilkan nilai jarak antar objek. Nilai jarak yang kecil menunjukkan bahwa dua data memiliki karakteristik yang mirip, sedangkan nilai jarak yang lebih besar menunjukkan bahwa kedua data memiliki perbedaan yang lebih signifikan.

  3. Distance Matrix

    Widget Distance Matrix digunakan untuk menampilkan hasil perhitungan jarak yang dihasilkan dari widget Distances dalam bentuk matriks. Setiap baris dan kolom pada matriks mewakili objek data, sedangkan nilai yang terdapat pada setiap sel menunjukkan jarak antara dua objek tersebut berdasarkan perhitungan Euclidean Distance.

    Melalui tampilan Distance Matrix, pengguna dapat dengan mudah melihat tingkat kemiripan antar data pada Iris Dataset. Data dengan nilai jarak yang kecil menunjukkan bahwa karakteristik antar bunga iris tersebut hampir sama, sedangkan nilai jarak yang lebih besar menunjukkan adanya perbedaan pada atribut bunga yang dianalisis. Visualisasi ini juga dapat membantu dalam memahami pola data serta menjadi dasar untuk analisis lanjutan seperti clustering atau pengelompokan data.

    Poin Utama Analisis:

    • Objek Data: Setiap baris dan kolom dalam matriks mewakili satu sampel bunga iris dari dataset.

    • Metode Perhitungan: Nilai jarak yang ditampilkan dihitung menggunakan metode Euclidean Distance.

    • Interpretasi Nilai:

      • Nilai Kecil (Warna Putih/Terang): Menunjukkan tingkat kemiripan yang tinggi. Karakteristik antar bunga iris tersebut hampir sama.

      • Nilai Besar (Warna Hijau): Menunjukkan adanya perbedaan signifikan pada atribut bunga yang dianalisis.

    • Kegunaan: Visualisasi ini membantu pengguna memahami pola data dan menjadi fondasi utama untuk analisis lanjutan, seperti clustering atau pengelompokan data.

B. Visualisasi Menghitung Jarak menggunakan Python#

Analisis juga dilakukan menggunakan Python dengan bantuan Google Colab. Dataset yang digunakan adalah Iris Dataset.

X = df[["sepal_length", "sepal_width", "petal_length", "petal_width"]].values

data_1 = X[0]
data_2 = X[1]

euclidean_distance = np.sqrt(np.sum((data_1 - data_2) ** 2))
manhattan_distance = np.sum(np.abs(data_1 - data_2))

print("Hasil Pehitungan jarak antar dua data :")
print(f"Euclidean Distance : {euclidean_distance:.4f}")
print(f"Manhattan Distance : {manhattan_distance:.4f}")

  • import numpy as np digunakan untuk mengimpor library NumPy untuk perhitungan numerik.

  • data_1 = X[0] mengambil data baris pertama.

  • data_2 = X[1] mengambil data baris kedua untuk dibandingkan.

Hasil perhitungan ditampilkan dalam bentuk nilai jarak yang menunjukkan seberapa mirip atau berbeda kedua data tersebut.

Berdasarkan analisis yang dilakukan menggunakan Orange Data Mining dan Python, dapat disimpulkan bahwa perhitungan jarak pada Iris Dataset dapat digunakan untuk mengetahui tingkat kemiripan antar data berdasarkan atribut numerik seperti sepal length, sepal width, petal length, dan petal width.
